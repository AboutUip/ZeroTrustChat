# MM1 安全内存模型技术规范

## 一、概述

MM1 是 ZChatIM 系统的安全核心，负责所有敏感数据的内存存储与处理。由 C++ 实现，仅通过 JNI 与 Spring Boot 交互。

**核心原则**：
- 消息不持久化，断电即失
- 宁可销毁不泄露
- 消息分散存储 + 独立加密

详见 [01-Overview.md](../01-Architecture/01-Overview.md)

**已实现 C++ 管理器（节选）**：`AuthSessionManager` → [02-Auth.md](../03-Business/02-Auth.md) §七；`SessionActivityManager` → [04-Session.md](../03-Business/04-Session.md) §七。

## 三、核心组件

### 3.1 Random Allocator

每次分配随机地址，物理不连续。

### 3.2 Address Mapper

加密索引：MessageID → {Address, KeySalt}

### 3.3 Key Manager

双棘轮密钥链：
```
IdentityKey → SessionKey → MessageKey(每条消息独立)
```

### 3.4 Message Manager

- 消息块独立加密
- 链式哈希（可选）
- 序列号排序

## 四、消息块结构

```
┌────────────────────────────────────────────────────┐
│  BlockHead (16B) │ CryptoKey(32B) │ Content │Auth  │
│                  │                │ (变长)   │(16B) │
└────────────────────────────────────────────────────┘

BlockHead:
┌─────────┬──────┬───────┬───────┬────────────┐
│Magic(2) │Ver(1)│Flag(1)│Size(4)│IDHash(8)   │
└─────────┴──────┴───────┴───────┴────────────┘

Content (明文，加密存储):
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│Sequence  │Timestamp │ PrevHash │ SenderID │ Payload  │
│  (8B)    │  (8B)    │  (32B)   │  (16B)   │ (变长)    │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

## 五、存储流程

1. 生成/获取消息密钥
2. 构造消息块 (Sequence + Timestamp + Payload)
3. 随机地址分配
4. 独立加密 (AES-256-GCM + 独立 Nonce)
5. 写入内存
6. 更新加密索引

## 六、安全机制

### 6.1 防 Dump

| 机制 | 说明 |
|------|------|
| 内存加密 | 存储时为密文 |
| 随机地址 | 每次运行地址不同 |
| 分散存储 | 消息物理不连续 |
| 反调试 | 检测调试器，触发自毁 |

### 6.2 自毁触发

| 场景 | 动作 |
|------|------|
| 用户登出 | Level 2 擦除 |
| 会话结束 | Level 2 擦除密钥 |
| 检测调试 | Level 3 物理销毁 |
| 收到异常信号 | Level 3 全量清零 |

### 6.3 销毁级别

| 级别 | 触发场景 | 覆写轮数 | 方法 |
|------|----------|:--------:|------|
| Level 1 | 常规内存释放 | 0 | 标记释放 |
| Level 2 | 登出/会话结束 | 1 | 固定值覆写 (0x00) |
| Level 3 | 调试检测/异常信号 | 3 | 随机数覆写 + mlock |

#### mlock 保护

- Level 3 必须使用 mlock 锁定内存
- 防止敏感数据被 swap 到磁盘
- 进程终止后自动释放

## 七、侧信道防护

### 7.1 常量时间实现

所有密码学比较必须使用常量时间算法。

### 7.2 Cache 攻击防护

| 攻击类型 | 防护措施 |
|----------|----------|
| Flush+Reload | 使用 clflush 指令清除敏感数据 |
| Evict+Reload | 避免在敏感操作中使用分支预测 |
| Prime+Probe | 使用常量时间查表实现 |

### 7.3 功耗分析防护

| 措施 | 说明 |
|------|------|
| 随机时钟插入 | 在密码学运算中插入随机延迟 |
| 功耗均衡 | 确保持续电流消耗 |
| 毛刺检测 | 检测异常电源波动 |

### 7.4 完整防护层

```
L1: 常量时间实现
    - 密钥比较
    - 签名验证
    - HMAC 计算

L2: Cache 隔离
    - 敏感数据立即清除
    - 常量时间查表
    - 内存屏障

L3: 运行时检测
    - 调试器检测
    - 虚拟机检测
    - 代码完整性校验
```

## 八、JNI 安全强化

### 8.1 三层防护体系

```
L1: 调用签名验证
    - Java 方法哈希校验
    - 调用者身份验证
    - 参数类型检查

L2: 参数校验
    - 边界检查
    - 空指针检查
    - 长度限制

L3: 内存隔离
    - 独立内存池
    - 边界保护
    - 溢出检测
```

### 8.2 参数校验矩阵

| 接口 | 参数 | 校验规则 |
|------|------|----------|
| storeMessage | sessionId | > 0 && < 0xFFFFFFFF |
| storeMessage | data | != nullptr && len <= 16MB |
| getMessage | msgId | != nullptr && len == 16 |
| storeKey | keyType | >= 0 && <= 4 |
| storeKey | keyData | != nullptr && len == 32 |
| emergencyWipe | 无 | 无 |
