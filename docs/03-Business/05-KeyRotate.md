# 密钥刷新技术规范

## 一、刷新机制

```
┌─────────────────────────────────────────────────────────────┐
│                    双密钥并行机制                              │
├─────────────────────────────────────────────────────────────┤
│  刷新期间: 新密钥 + 旧密钥 同时生效                              │
│  发送:   使用新密钥加密                                        │
│  接收:   新密钥/旧密钥均可解密                                  │
└─────────────────────────────────────────────────────────────┘
```

## 二、刷新流程

```
定时刷新 (每24小时):
1. MM1 生成新密钥
2. 新密钥立即生效
3. 旧密钥保留 24小时
4. 24小时后旧密钥失效

消息处理:
- 发送方: 始终使用最新密钥
- 接收方: 同时支持新旧密钥解密
```

## 三、密钥状态

| 阶段 | 新密钥 | 旧密钥 |
|------|--------|--------|
| 刷新前 | 未生成 | 生效中 |
| 刷新中 | 已生成 | 生效中 |
| 刷新后 | 生效中 | 保留中 |
| 24h后 | 生效中 | 失效 |

## 四、安全保证

- 生成: MM1 可信区
- 存储: 仅存内存
- 销毁: Level 2
- 服务重启: 密钥丢失
