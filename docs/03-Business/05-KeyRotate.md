# 密钥刷新技术规范

## 一、分级轮换机制

### 1.1 密钥层级

| 密钥类型 | 轮换周期 | 说明 |
|----------|:--------:|------|
| Identity Key | 24小时 | 长期身份密钥 |
| Master Key | 6小时 | 主密钥 |
| Session Key | 1小时 | 会话密钥 |
| Message Key | 每条消息 | 消息密钥 |

### 1.2 轮换时序

```
IdentityKey(24h) ─────────────────────────────────────────────────
MasterKey(6h)    ──── ──── ──── ──── ──── ──── ──── ──── ──── ────
SessionKey(1h)  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─
MessageKey      X   X   X   X   X   X   X   X   X   X   X   X   X
```

## 二、双密钥并行

刷新期间新旧密钥同时生效：

| 阶段 | 新密钥 | 旧密钥 |
|------|--------|--------|
| 刷新前 | 未生成 | 生效中 |
| 刷新中 | 已生成 | 生效中 |
| 刷新后 | 生效中 | 保留1小时 |
| 1小时后 | 生效中 | 失效 |

## 三、强制触发

以下情况强制触发密钥刷新：

- 用户登出
- 会话结束
- 检测到异常行为
- 手动触发

## 四、后向保密设计

### 4.1 双重保密机制

```
前向保密: 消息密钥泄露不影响历史消息
后向保密: 密钥泄露不影响未来消息
```

### 4.2 B级: 消息级 DH 棘轮

| 参数 | 值 |
|------|-----|
| DH 棘轮触发间隔 | 每100条消息 |
| 触发时机 | 发送第100、200、300...条消息时 |

### 4.3 C级: Identity Key 定期更换

| 参数 | 值 |
|------|-----|
| Identity Key 更换周期 | 24小时 |
| 更换时保留旧密钥 | 72小时 |
| 保留期间解密能力 | 新旧密钥均可 |

### 4.4 后向保密状态机

```
[初始化] ──▶ [前向保密模式]
                │
                ├── 每100条消息 ──▶ [DH棘轮触发]
                │                   - 更换发送方DH密钥对
                │                   - 派生新根密钥
                │                   - 重置消息计数
                │
                ├── 24小时到期 ──▶ [IdentityKey更换]
                │                   - 生成新IdentityKey
                │                   - 广播公钥给会话方
                │                   - 旧IdentityKey保留72h
                │
                └── [完整的Future Secrecy保障]
```

### 4.5 安全保证

| 威胁场景 | 保护能力 |
|----------|----------|
| 当前消息密钥泄露 | 仅影响当前消息 |
| 会话密钥泄露 | 仅影响100条消息 |
| Identity Key泄露 | 仅影响24小时内的未来消息 |
| 完整会话泄露 | 影响范围 ≤ 100条 + 72h |

## 五、安全保证

- 生成: MM1 可信区
- 存储: **协议层密钥链（Identity / Master / Session 等）以 MM1 内存叙述为主**（见 **`01-MM1.md`** / 本节流程）
- **与 MM2 区分**：**消息 AES 密钥文件** **`indexDir/mm2_message_key.bin`** **落盘**，进程重启后仍在（除非 **`Cleanup` / `CleanupAllData` / 删目录**）；见 **`03-Storage.md` 第三节**、**`docs/README.md`**「冲突与权威」
- 销毁: Level 2（内存与策略性覆写）
- 服务重启: **MM1 内存态密钥与会话丢失**；**`mm2_message_key.bin` 不因此自动丢失**
