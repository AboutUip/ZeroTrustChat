# 认证安全机制技术规范

## 一、限流规则

```
MM1 认证计数器:
IP级别: 5次/分钟
用户级别: 10次/分钟
```

## 二、封禁机制

### 2.1 动态封禁策略

封禁时长根据失败次数动态计算：

```
封禁时长 = min(失败次数 × 基础时长, 最大封禁时长)
```

| 参数 | 值 | 说明 |
|------|-----|------|
| 基础时长 | 15分钟 | 每次失败累加 |
| 最大时长 | 24小时 | 封禁上限 |
| 封禁窗口 | 24小时 | 窗口期内连续计算 |
| 失败计数阈值 | 5次 | 触发封禁 |

### 2.2 封禁矩阵

| 连续失败次数 | 封禁时长 |
|:-------------:|:--------:|
| 5 | 15分钟 |
| 6 | 30分钟 |
| 7 | 45分钟 |
| 8 | 1小时 |
| 9 | 1小时15分钟 |
| 10 | 1小时30分钟 |
| 11+ | 最大24小时 |

### 2.3 封禁存储

| 项目 | 要求 |
|------|------|
| 存储位置 | MM1 内存 |
| 索引键 | userId + IP |
| 重启丢失 | 是 |
| 日志记录 | 否 |

### 2.4 解封机制

| 方式 | 说明 |
|------|------|
| 自动解封 | 封禁时长到期自动解除 |
| 手动解封 | 管理员强制解除 |
| 重启不解封 | 服务重启保持封禁状态 |

## 三、错误码

| 错误码 | 说明 |
|--------|------|
| E_AUTH_RATE_LIMIT | 频率超限 |
| E_AUTH_LOCKED | 已封禁 |
| E_AUTH_FAILED | 认证失败 |

## 四、认证成功

```
成功 → 重置MM1计数器 → 返回sessionId
```

## 五、安全原则

- 计数器/封禁存MM1内存
- 服务重启 → 状态丢失
- 不记录任何日志
- Level 2 销毁

## 六、参数汇总

| 参数 | 值 |
|------|-----|
| IP级限流 | 5次/分钟 |
| 用户级限流 | 10次/分钟 |
| 封禁基础时长 | 15分钟 |
| 封禁最大时长 | 24小时 |
| 封禁窗口 | 24小时 |
| 失败计数阈值 | 5次 |
