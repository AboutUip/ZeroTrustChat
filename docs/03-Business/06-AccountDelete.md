# 账户注销技术规范

## 一、注销流程

```
用户发起注销:
1. 身份验证 (重新认证)
2. 确认注销 (二次验证)
3. 执行注销 (不可逆)
```

## 二、执行步骤

```
1. 验证身份:
   - 要求用户重新输入密码
   - 验证 session 有效性

2. 密钥销毁 (Level 3):
   - 销毁 MM1 中所有密钥
   - 标记用户状态为"已注销"
   - 清除好友关系
   - 清除群组成员资格

3. 保留数据:
   - 保留 .zdb 文件 (加密状态)
   - 保留消息缓存 (无法解密)
```

## 三、安全机制

```
设计原则:
- 不删除任何数据
- 仅销毁密钥
- 物理获取 .zdb → 无法解密 → 不知道是否注销

验证:
- 必须重新认证
- 需要二次确认

销毁:
- Level 3 级别
- MM1 密钥覆写
- 标记"已注销"
```

## 四、服务器处理

```
收到注销请求:
1. 验证 session 和密码
2. MM1 执行 Level 3 销毁
3. 标记用户为"已注销"
4. 保留所有数据 (无法解密)
```

## 五、数据处理

| 数据类型 | 处理方式 | 说明 |
|----------|----------|------|
| .zdb 文件 | 保留 | 加密状态，无法解密 |
| 消息缓存 | 保留 | 无密钥，无法解密 |
| 密钥 | Level 3 销毁 | 物理覆写 |
| 好友关系 | 标记清除 | 逻辑删除 |
| 群组成员 | 标记清除 | 逻辑删除 |
| 会话 | Level 2 销毁 | 清除 |

## 六、访问控制

```
已注销用户:
- 登录: 永久禁止
- 数据访问: 无法解密
- 物理获取: 无法判断是否注销
```
