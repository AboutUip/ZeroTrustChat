# ZChatIM 技术方案文档

## 一、架构

```
┌─────────┐      ZSP       ┌─────────────┐      JNI       ┌────────┐
│  客户端  │ ───────────> │  SpringBoot │ ───────────> │   C++   │
│         │ <──────────  │  (Netty)    │ <────────── │ (MM1/MM2)│
└─────────┘               └─────────────┘               └────┬─────┘
                                                              │
                                                              ▼
                                                     ┌────────────────┐
                                                     │   data/        │
                                                     │   *.zdb (500KB)│
                                                     └────────────────┘
```

---

## 二、模块职责

### SpringBoot (不可信区)

- ZSP 协议编解码
- 消息路由
- 业务调度
- 调用 JNI

### C++ (可信区)

- 消息加密/解密
- 密钥管理
- .zdb 文件存储
- 安全销毁

---

## 三、存储机制

### .zdb 文件

```
目录: data/
文件: [随机8字符].zdb (500KB)
数量: 5-2000
填充: 创建时随机噪声填满
写入: 随机位置 + 加密
```

### SQLite

```
表: zdb_files, data_blocks, user_data, group_data, group_members
密钥: 由 MM1 管理
操作: 仅 C++ 可访问
```

---

## 四、数据分类

| 数据 | 持久化 | 过期 |
|------|--------|------|
| 用户元数据 | .zdb | 不过期 |
| 好友列表 | .zdb | 不过期 |
| 群组信息 | .zdb | 不过期 |
| 聊天消息 | 内存 | 7天 |
| 文件数据 | 内存 | 7天 |
| 会话密钥 | 内存 | 24小时 |

---

## 五、推送机制

```
在线: 用户A → 服务器 → 用户B
离线: 用户B上线 → 同步请求 → 服务器推送未同步消息
```

---

## 六、安全机制

```
销毁级别:
- Level 1: 标记释放
- Level 2: 覆写
- Level 3: mlock + 多轮覆写

触发: 登出/会话结束/检测调试/异常信号
```

---

## 七、密钥刷新

```
周期: 24小时
密钥: Master Key, 文件加密密钥, 用户长期密钥
```

---

## 八、服务重启

```
丢弃: 消息、文件、会话密钥
保留: 用户元数据、好友、群组
```
