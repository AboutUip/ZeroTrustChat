# 灾备恢复技术规范

## 一、设计原则

```
安全 > 可用性
零备份 → 零泄露入口
```

## 二、服务恢复

```
崩溃恢复:
1. 扫描 .zdb 加载元数据
2. 丢弃: 消息/会话密钥/MM1状态
3. 客户端重连 → 重新认证
```

## 三、数据状态

| 数据 | 崩溃后 |
|------|--------|
| 用户元数据 | 保留 (.zdb) |
| 好友列表 | 保留 (.zdb) |
| 群组信息 | 保留 (.zdb) |
| 聊天消息 | 丢失 |
| 会话密钥 | 丢失 |

## 四、客户端处理

```
- 不缓存消息
- 自动重连
- 重连后 SYNC 拉取 (空)
- 提示用户消息已清除
```

## 五、安全性

- 无备份文件
- 无恢复接口
- 服务重启 → Level 2 销毁
