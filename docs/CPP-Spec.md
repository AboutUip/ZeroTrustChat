## 三、存储机制

### 3.1 .zdb 文件

```
目录: data/
文件: [随机8字符].zdb (500KB)
数量: 5-2000 个
填充: 创建时用随机噪声填满
```

### 3.2 数据存储

```
写入流程:
1. 随机选择 .zdb 文件
2. 随机选择空闲位置
3. 检查防覆盖
4. 写入加密数据
5. 记录到 SQLite
```

### 3.3 并发控制

```
方案: 文件锁 + SQLite 事务
- 写操作: 加写锁 + BEGIN EXCLUSIVE
- 读操作: 加读锁
- 粒度: 每个文件独立锁
```

## 四、数据持久化

### 4.1 持久化数据

| 数据类型 | 存储位置 |
|----------|----------|
| 用户元数据 | .zdb |
| 好友列表 | .zdb |
| 群组信息 | .zdb |
| 文件密钥 | .zdb (加密) |
| 聊天消息 | 内存 (7天) |
| 会话密钥 | 内存 |

### 4.2 SQLite 表

```sql
zdb_files: file_id, total_size, used_size
data_blocks: block_id, data_id, chunk_idx, file_id, offset, length, sha256
user_data: user_id, file_id, offset, length, sha256, type
group_data: group_id, file_id, offset, length, sha256
group_members: group_id, user_id, role, joined_at
```

## 五、密钥管理

### 5.1 密钥层级

```
Master Key → 文件加密密钥 → 用户密钥
```

### 5.2 刷新机制

```
周期: 24 小时
流程: 生成新密钥 → 重新加密数据 → 更新存储 → 销毁旧密钥
```

## 七、推送机制

### 7.1 在线推送

```
用户A 发送 → 服务器 → 用户B (在线)
```

### 7.2 离线同步

```
用户B 上线:
1. 客户端建立 ZSP 连接
2. 客户端发送同步请求 (最后消息ID)
3. 服务器推送未同步消息
4. 客户端接收并显示
```

## 八、安全机制

### 8.1 销毁级别

| 级别 | 动作 |
|------|------|
| Level 1 | 标记释放 |
| Level 2 | 覆写 0x00/随机数 |
| Level 3 | mlock + 多轮覆写 |

### 8.2 触发条件

| 场景 | 级别 |
|------|------|
| 用户登出 | Level 2 |
| 会话结束 | Level 2 |
| 检测调试 | Level 3 |
| 异常信号 | Level 3 |

|------|------|------|

## 十、服务重启

```
1. 加载密钥
2. 扫描持久化数据
3. 丢弃: 消息、文件、会话密钥
4. 重建内存索引
5. 等待客户端重连
```
