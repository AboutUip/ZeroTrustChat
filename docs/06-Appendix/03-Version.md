# 协议版本兼容性技术规范

> **与实现**：ZSP 帧头 **1 字节版本**见 **`docs/01-Architecture/02-ZSP-Protocol.md`**（**第六节 / Header** 与 **`03-Version` 第五节** 示意一致）。**`ZChatIM`** 若增加独立 **ABI/架构版本**，须在 **`Types.h` / 握手报文** 中单独定义，并回写本节。  
> **C++ 与 Java**：**`ZChatIMCore`** 版本与 **客户端 App version** 协商不在本仓库自动完成；**`05-ZChatIM-Implementation-Status.md`** 描述 native 能力边界。

## 一、版本管理

```
版本格式: major.minor.patch
例如: 1.0.0

major: 重大变更，不兼容
minor: 新功能，向后兼容
patch: bug修复，向后兼容
```

## 二、版本协商

```
客户端连接流程:
1. 客户端发送 HELLO
2. 携带客户端版本: client_version
3. 服务器返回 SERVER_HELLO
4. 携带服务器版本: server_version
5. 协商最终版本: min(client.major, server.major)

版本兼容:
- major相同: 完全兼容
- major不同: 不兼容，断开连接
```

## 三、版本降级

```
服务器不支持客户端版本:
1. 返回 ERROR_VERSION_MISMATCH
2. 携带支持的最新版本
3. 建议客户端升级
4. 断开连接

客户端不支持服务器版本:
1. 提示用户升级客户端
2. 断开连接
```

## 四、功能开关

```
基于版本启用功能:
┌─────────────────────────────────────────────────────────────┐
│  功能                       │  v1.0.0  │  v1.1.0  │  v2.0.0  │
├─────────────────────────────────────────────────────────────┤
│  基础消息                   │    ✓     │    ✓     │    ✓     │
│  文件传输                   │    ✓     │    ✓     │    ✓     │
│  消息编辑                   │    ✗     │    ✓     │    ✓     │
│  消息回复                   │    ✗     │    ✗     │    ✓     │
│  群禁言                     │    ✗     │    ✗     │    ✓     │
└─────────────────────────────────────────────────────────────┘
```

## 五、版本检测

```
ZSP Header:
┌─────────┬──────┬───────┬───────┬────────────┐
│Magic(2) │Ver(1)│Flag(1)│Size(4)│IDHash(8)   │
└─────────┴──────┴───────┴───────┴────────────┘
          ↑
       版本号(1B)
```

## 六、错误码

| 错误码 | 说明 |
|--------|------|
| E_VERSION_MISMATCH | 版本不兼容 |
| E_VERSION_DEPRECATED | 版本已废弃 |
| E_UPGRADE_REQUIRED | 需要升级 |

## 七、升级策略

```
客户端:
- 启动时检查版本
- 弹窗提示升级
- 阻止过低版本登录

服务器:
- 维护支持版本列表
- 平滑过渡旧版本
- 定期废弃旧版本
```
