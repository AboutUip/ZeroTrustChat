# 证书固定技术规范

## 一、固定方式

```
┌─────────────────────────────────────────────────────────────┐
│  公钥哈希固定 (SPKI SHA-256)                                  │
├─────────────────────────────────────────────────────────────┤
│  当前证书公钥哈希                                              │
│  备用证书公钥哈希                                              │
└─────────────────────────────────────────────────────────────┘
```

## 二、验证流程

```
1. TLS握手获取服务器证书
2. 提取公钥 → SHA-256
3. 与内置哈希比对
4. 不匹配 → 断开连接 → 标记异常
5. 连续3次异常 → 封禁客户端
```

## 三、自动轮换

```
服务端:
├── Active Certificate (当前)
└── Standby Certificate (备用)

轮换: Active → Standby，新证书生效
```

## 四、异常封禁

| 项目 | 值 |
|------|-----|
| 触发 | 连续3次证书验证失败 |
| 存储 | MM1内存 |
| 解除 | 管理员手动 |

## 五、客户端内置

- 当前公钥哈希
- 备用公钥哈希
- 算法: SHA-256
