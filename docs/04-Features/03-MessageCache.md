# 消息缓存上限技术规范

## 一、缓存策略

```
┌─────────────────────────────────────────────────────────────┐
│  MM2 消息缓存                                                │
├─────────────────────────────────────────────────────────────┤
│  限制: 100条/会话                                            │
│  淘汰: LRU (仅未读消息)                                       │
│  过期: 7天                                                   │
│  已读: 不参与淘汰                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、会话队列

```
┌─────────────────────────────────────────────────────────────┐
│  per_session_message_queue                                  │
├─────────────────────────────────────────────────────────────┤
│  sessionId → deque<msgId> (max 100)                         │
│                                                             │
│  LRU淘汰 (仅未读消息):                                        │
│  - 未读消息超过100条 → 淘汰最旧未读                             │
│  - 已读消息不参与淘汰                                          │
│  - 消息被读取 → 标记为已读                                     │
└─────────────────────────────────────────────────────────────┘
```

## 三、过期机制

```
三重机制:
1. 未读: 100条上限 (LRU淘汰最旧)
2. 已读: 不淘汰，永久保留直到读取
3. 时间: 7天 (未读消息)

服务重启:
- 所有缓存丢失
```

## 四、参数

| 项目 | 值 |
|------|-----|
| 未读上限 | 100条/会话 |
| 淘汰规则 | LRU (仅未读) |
| 已读处理 | 不淘汰 |
| 过期时间 | 7天 |
| 存储 | MM2 内存 |
