# 消息同步机制技术规范

## 一、设计原则

- 无离线/在线消息之别
- 消息统一存MM2，MM1仅存索引
- 服务重启消息即失
- 客户端不缓存，每次连接拉取

## 二、架构

```
MM1: 索引层
     └── MessageID → {MM2_Address, KeyID, Timestamp}

MM2: 存储层
     └── 随机地址 + 加密消息块 (AES-256-GCM)
```

## 三、同步流程

```
客户端连接:
1. ZSP握手
2. 客户端发送 SYNC(lastMsgId / timestamp)
3. 服务器从MM2拉取该时间点后的消息
4. 依次推送
5. 7天前消息过期销毁
```

## 四、消息结构

```
┌─────────────────────────────────────────────────────────────┐
│  MM2 消息块                                                  │
├─────────────────────────────────────────────────────────────┤
│  BlockHead (16B) │ CryptoKey(32B) │ Content │ Auth Tag(16B) │
└─────────────────────────────────────────────────────────────┘
```

## 五、服务重启

| 数据 | 保留 | 丢失 |
|------|------|------|
| 用户元数据 | ✓ | - |
| 好友列表 | ✓ | - |
| 群组信息 | ✓ | - |
| 聊天消息 | - | ✓ |
| 会话密钥 | - | ✓ |

## 六、容量

- 单用户消息缓存: 100条 (超出淘汰最旧)
- 全局消息过期: 7天
